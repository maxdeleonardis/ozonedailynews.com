┌─────────────────────────────────────────────────────────────────────────────┐
│                    FAILURE SCENARIOS (All Handled)                           │
└─────────────────────────────────────────────────────────────────────────────┘

Scenario 1: Files moved without registry update
├─ Layer 1 (Pre-build): ❌ Fails validation, blocks deployment
└─ Layer 4 (Runtime):   ✅ Recursive fallback finds file anyway

Scenario 2: Registry has wrong filePath
├─ Layer 1 (Pre-build): ❌ Detects file doesn't exist at path
└─ Layer 4 (Runtime):   ✅ Recursive fallback finds correct location

Scenario 3: Slug stored with leading slash in DB
├─ Layer 3 (Link gen):  ✅ Strips slash before generating URL
└─ Layer 2 (Lookup):    ✅ Tries multiple slug variants

Scenario 4: Registry corrupted/deleted
├─ Layer 1 (Pre-build): ⚠️  Warning but doesn't block
└─ Layer 4 (Runtime):   ✅ Recursive search finds all files

Scenario 5: New sharding structure added
├─ Layer 1 (Pre-build): ✅ Validates after sync-registry
└─ Layer 2 (Lookup):    ✅ Registry has new paths automatically

Scenario 6: Double-slash URL generated
├─ Layer 3 (Link gen):  ✅ Prevented by .replace(/^\//, '')
└─ Layer 1 (Pre-build): ✅ Validated as "properly formatted"
